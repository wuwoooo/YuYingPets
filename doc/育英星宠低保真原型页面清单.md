# 育英星宠低保真原型页面清单

版本：V1.0  
日期：2026-03-30  
适用文档：[`育英星宠PRD`](./育英星宠PRD.md)

## 一、说明

本文件用于定义 `育英星宠` 第一阶段低保真原型范围，目标是让产品、设计、前端能够围绕同一套页面骨架推进工作。

本次低保真原型分为两端：
- `Admin` 管理后台
- `Display` 班级展示端

原则：
- 先确定结构、模块、层级、交互关系
- 暂不讨论高保真视觉细节
- 重点保证“学校展示感 + 班级激励闭环 + 后台管理效率”

## 三、Admin 端低保真页面清单

## A01 Admin 登录页

### 页面目标

提供学校管理员、班主任、教师进入后台的统一入口，并建立品牌第一印象。

### 页面结构

- 左侧：学校品牌视觉区
- 右侧：登录表单区

### 核心模块

- 学校 logo / 育英星宠 logo
- 产品名称与副标题
- 账号输入框
- 密码输入框
- 登录按钮
- 忘记密码/帮助入口

### 交互说明

- 输入账号密码后登录
- 登录成功后根据角色进入不同后台默认页

### 原型重点

- 虽然是后台登录页，也要体现学校定制系统身份
- 不做通用 SaaS 登录页

## A02 Admin 首页 / 校级驾驶舱

### 页面目标

作为整个系统最重要的后台首页，用于展示学校级关键成果与运营状态。

### 页面结构

- 顶部：全局品牌栏
- 左侧：主导航
- 中央：驾驶舱主内容区
- 右侧或底部：预警/公告/快捷入口区

### 核心模块

- 顶部信息栏
- 核心指标卡区
- 年级/班级分布区
- 趋势分析区
- 高光榜单区
- 荣誉成果区
- 重点预警区
- 快捷操作区

### 详细模块说明

#### 1. 顶部信息栏

包含：
- 学校 logo
- 产品名 `育英星宠`
- 当前学期
- 当前日期
- 用户信息
- 快捷切换入口

#### 2. 核心指标卡区

建议展示：
- 全校总积分
- 活跃班级数
- 活跃学生数
- 本周正向行为次数
- 勋章发放数
- 平均成长等级

#### 3. 年级/班级分布区

建议包含：
- 年级活跃度对比
- 班级积分分布
- 学段参与率

#### 4. 趋势分析区

建议包含：
- 近 7 天成长趋势
- 近 30 天行为趋势
- 阶段活动趋势

#### 5. 高光榜单区

建议包含：
- 明星班级
- 明星学生
- 荣誉勋章榜

#### 6. 荣誉成果区

建议包含：
- 最新荣誉获得情况
- 集体荣誉展示
- 阶段成果海报位

#### 7. 重点预警区

建议包含：
- 低活跃班级
- 长时间无更新班级
- 异常数据提醒

### 交互说明

- 顶部筛选支持按学段、年级、班级切换
- 指标卡、图表卡可点击进入详情页

### 原型重点

- 这页不是“管理首页”，而是“校级汇报首页”
- 要确保布局具有主次关系和展示感

## A03 班级管理页

### 页面目标

管理学校内班级的基础信息、班主任、展示配置。

### 页面结构

- 顶部：筛选与操作栏
- 中部：班级列表
- 侧边或弹窗：班级编辑表单

### 核心模块

- 状态筛选
- 新建班级按钮
- 班级列表表格/卡片
- 编辑弹窗

### 列表字段建议

- 班级名称
- 年级
- 班主任
- 学生人数
- 班级目标

- 支持配置 Display 展示参数

### 原型重点

- 保持后台效率
- 给后续 Display 绑定班级提供基础配置能力

## 四、Display 端低保真页面清单

## D01 Display 进入页

### 页面目标

承担 Display 端待机展示与进入前品牌引导。

### 页面结构

- 顶部：校训
- 中央：校徽 + 产品名 + 副标题
- 下方：班主任登录入口
- 底部：教学楼线稿背景

### 核心模块

- `display-bg.png` 背景
- 产品名 `育英星宠`
- 副标题
- 登录按钮
- 待机提示语

### 页面文案建议

- 主标题：`育英星宠`
- 副标题：`育英学校班级成长激励系统`
- 提示：`请班主任登录，开启班级星宠成长之旅`

### 原型重点

- 这页必须体现“学校主视觉入口”，不是普通登录页

## D02 班主任登录页

### 页面目标

让班主任快速进入班级展示端。

### 页面结构

- 中央登录面板
- 背景仍延续学校主视觉

### 核心模块

- 账号输入
- 密码输入
- 登录按钮
- 返回首页按钮

### 原型重点

- 输入面板要简洁，避免破坏主视觉

## D03 进入校园/教学楼/班级动画过场页

### 页面目标

让 Display 端拥有明确的进入仪式感。

### 页面分镜

#### 第一屏：校园待机

- 展示背景图
- 校训、校徽、产品名静态呈现

#### 第二屏：进入校园

- 中央品牌区淡出
- 镜头沿阶梯向前推进
- 教学楼线稿逐渐放大

#### 第三屏：进入班级

- 班级名称出现
- 班主任名称出现
- 班级口号短暂展示

#### 第四屏：转入主页

- 学生萌宠卡片逐步进入
- 榜单和班级信息就位

### 原型重点

- 这一页在低保真里至少要画出分镜切换关系

## 五、页面间关系

## Admin 端主流程

1. 登录后台
2. 进入校级驾驶舱
3. 进入班级/学生/规则等管理页
4. 进入数据分析与系统设置

## Display 端主流程

1. 进入页待机展示
2. 班主任登录
3. 播放进入校园/教学楼/班级动画
4. 进入班级主页
5. 轮播榜单页、荣誉页、目标页

## 六、低保真原型输出建议

建议设计阶段按以下顺序出图：

1. `D01 Display 进入页`
2. `D03 动画分镜页`
3. `D04 Display 班级主页`
4. `D05 榜单页`
5. `A02 校级驾驶舱`
6. `A03 班级管理页`
7. `A04 学生管理页`
8. `A05 积分规则管理页`

原因：
- 先把品牌入口和班级展示端确定，能快速建立项目气质
- 再补后台核心页，保证产品骨架成立

## 七、结论

低保真阶段最重要的不是把所有页都画满，而是先确认这几个核心问题：

- `Display` 进入页是否足够有学校仪式感
- 班级主页是否足够适合白板展示
- 榜单与荣誉是否足够有舞台感
- `Admin` 首页是否足够支撑校级汇报

只要这四个问题先定下来，`育英星宠` 的产品骨架就会非常稳。
