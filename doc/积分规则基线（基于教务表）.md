# 积分规则基线（基于教务表）

来源文件：[`加扣分-教务模块（课堂、作业、学业）.xls`](/Users/wuwoo/Desktop/work/_育英星宠/YuYingPets/doc/加扣分-教务模块（课堂、作业、学业）.xls)

## 1. 用途

本文件用于把 Excel 规则表转成开发可用的规则基线，供以下模块参考：

- 积分规则管理
- Display / Admin 加减分操作
- 学生行为标签沉淀
- AI 学情评估结构化输入

## 2. 当前确认的规则来源结构

该表包含以下工作表：

- 教务
- 语文组
- 数学组
- 英语组
- 物理组
- 化学组
- 地理组
- 生物组
- 历史组
- 政治组
- 音美信综合组
- 体育组

说明：

- `教务` 可视为通用规则
- 其余工作表可视为学科规则

## 3. 通用教务规则示例

正向：

- 课堂表现优秀：`+2`
- 作业优秀：`+2`
- 考试进步显著：`+5`
- 学科竞赛获奖：`+5 ~ +10`
- 优秀学习小组：`+3`
- 早读认真：`+1`
- 上课坐姿端正：`+2`
- 上课期间桌面收纳整齐：`+2`

负向：

- 上课迟到：`-2`
- 课堂违纪：`-2`
- 缺交/迟交作业：`-2`
- 作业质量差：`-2`
- 考试作弊：`-10`
- 扰乱课堂秩序：`-3`
- 自习课纪律差：`-2`
- 无故旷课：`-5`
- 学习态度不端正：`-2`
- 上课坐姿不端正：`-2`

## 4. 学科规则特征

从各学科表可以总结出几类高频规则场景：

- 课堂表现：专注、迟到、违纪、打瞌睡、展讲、答疑
- 作业表现：提前交、未交、优秀、不合格、抄袭、书写
- 测评表现：周测、月评价、期末考核、听默写
- 背诵与早读：按时完成、听默写、背书、早读状态
- 学科活动：竞赛获奖、活动参与、技能展示

## 5. 对规则建模的直接要求

规则表至少需要支持：

- 通用规则与学科规则并存
- 正向/负向行为标记
- 固定分值
- 区间分值
- 场景标签
- 行为维度标签

建议字段：

- `module_type`
- `subject_code`
- `scene_code`
- `rule_name`
- `score_mode`
- `score_value`
- `min_score`
- `max_score`
- `sentiment`
- `dimension`
- `tag`
- `description`
- `ai_summary_text`

## 6. 对前端交互的影响

Display / Admin 加减分不能只做“+1 / -1”。

需要支持两类操作：

- 快捷固定项
  - 如：课堂表现优秀、作业优秀、课堂违纪
- 学科规则项
  - 如：数学周测进步、英语听默写全对、体育比赛获奖

建议前端规则选择结构：

- 一级：通用 / 学科
- 二级：学科
- 三级：场景
- 四级：规则项

Display 端为了效率，可以只显示：

- 班主任/任课教师所在学科的高频规则
- 班级通用高频规则

Admin 端则显示完整规则树。

## 7. 对 AI 学情评估的影响

规则表中的每条记录，不应只记为“加了几分”，还应转换为行为信号，例如：

- `课堂表现优秀` -> `课堂学习 / 专注投入 / 正向`
- `缺交作业` -> `作业管理 / 任务完成 / 负向`
- `考试进步显著` -> `学业成绩 / 成长进步 / 正向`
- `课堂违纪` -> `课堂纪律 / 自我管理 / 负向`

这样 AI 才能输出真正有价值的阶段总结。

## 8. 第一阶段实施建议

第一阶段建议这样落地：

1. 先把通用教务规则和主要学科规则导入数据库
2. 把所有区间规则先拆成后台可选固定档位
3. Display 端只放高频规则
4. Admin 端支持完整规则查看和配置
5. 学情评估先读取结构化标签统计，不直接读原始表述
