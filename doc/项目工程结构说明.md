# 育英星宠项目工程结构说明

## 1. 目标结构

项目正式按三端拆分：

```text
YuYingPets/
├─ backend/          后端服务
├─ admin-web/        管理后台前端
├─ display-app/      Display 独立应用
│  ├─ shared/        Display 共享业务层
│  ├─ web/           Display 网页版入口
│  └─ desktop/       Display Windows 桌面壳
└─ doc/              产品、技术、接口文档
```

## 2. 拆分原则

### 2.1 backend

负责：

- 鉴权与权限
- 班级、学生、积分、萌宠、奖励、AI 学情
- WebSocket 实时广播
- 管理后台与 Display 的统一数据源

### 2.2 admin-web

负责：

- 学校、班级、学生、规则、奖励、宠物配置
- 数据看板与汇报展示模式
- 教师与管理员日常操作

特点：

- 只作为 Web 应用交付
- 使用 React + 组件库 + 定制主题

### 2.3 display-app

负责：

- 学校进入页
- 班级主页
- 排行榜
- 兑换中心
- 白板场景快捷加减分

特点：

- 业务代码独立于 Admin
- 一套 React Display 代码输出两种形态

## 3. Display 双形态方案

### 3.1 Web 形态

部署方式：

- HTTP 网页
- 浏览器全屏运行
- 适用于白板浏览器、触控一体机浏览器、普通局域网访问

目录：

- `display-app/web`

### 3.2 Desktop 形态

部署方式：

- Windows 安装程序
- 本地安装到独立大屏终端
- 后续支持开机启动、固定窗口、系统级控制

目录：

- `display-app/desktop`

建议：

- 第一阶段使用 `Electron`

原因：

- Windows 打包和分发成熟
- 更适合大屏、kiosk、自动启动、窗口控制场景

### 3.3 Shared 层

目录：

- `display-app/shared`

负责放置：

- React 页面
- Display 路由
- Zustand 状态
- API Client
- WebSocket Client
- 页面组件、业务组件、主题变量

## 4. 当前落地状态

### 4.1 已完成

- 原根目录后端已迁移到 `backend`
- `admin-web` 已建立独立目录
- `display-app/shared`、`display-app/web`、`display-app/desktop` 已建立独立目录

### 4.2 下一步开发顺序

建议顺序：

1. 完成 `admin-web` Vite + React 基础初始化
2. 完成 `display-app/shared` 共享业务层
3. 完成 `display-app/web` 网页入口
4. 完成 `display-app/desktop` Electron 壳
5. 联调 `backend`

## 5. 注意事项

- `display-app` 不是两套前端，而是一套独立 Display 业务代码、两种交付壳
- `admin-web` 与 `display-app` 不应互相混用页面与路由
- 后续接口文档、部署文档、启动说明都应使用新路径，不再使用旧的根目录后端路径
