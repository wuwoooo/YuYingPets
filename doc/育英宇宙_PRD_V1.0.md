## 1. 产品结论

### 1.1 产品名称

**育英宇宙（YuyingVerse）**

### 1.2 一句话定义

育英宇宙是在育英星宠基础上升级的校园 2D 元世界入口，把学生评价、星宠成长、荣誉激励、AI 学情、数字教师、校园服务与家校展示统一到一个可探索的数字校园空间中。

### 1.3 产品定位

育英宇宙不是重新开发一套独立系统，而是育英星宠的体验层与业务域扩展：

- 以现有 `backend` 作为统一数据源。
- 复用现有 `admin-web` 作为配置、运营、审核与数据管理后台。
- 复用并扩展 `display-app` 的大屏展示能力。
- 新增学生/家长可访问的 `verse-web` 体验端，承载 2D 校园元世界。
- 新增少量业务域：校园地图、任务、内容、校长信箱、数字教师、家长绑定/查看。

最终产品形态是：

```text
育英星宠：评价 -> 积分 -> 星宠 -> 榜单 -> 荣誉 -> 数据
育英宇宙：数字身份 -> 2D 校园 -> 任务/服务 -> 成长反馈 -> 家校共看 -> AI 辅助
```

### 1.4 产品愿景

让每一个孩子拥有自己的数字生命。  
让每一位老师沉淀自己的数字教师。  
让每一座校园拥有可持续运营的教育元世界。

## 2. 当前基础盘点

### 2.1 已有工程架构

当前活跃仓库为 `/Users/wuwoo/Desktop/work/_育英星宠/YuYingPets`，现有工程已经拆为三部分：

| 工程 | 技术/形态 | 当前职责 | 育英宇宙中的定位 |
|---|---|---|---|
| `backend` | NestJS + Prisma + MySQL + WebSocket + Redis/BullMQ | 鉴权、班级、学生、积分、星宠、奖励、荣誉、AI 学情、Display 数据 | 继续作为唯一后端与统一业务源 |
| `admin-web` | React Web SPA | 管理后台、教师工作台、数据看板、配置管理 | 扩展为育英宇宙运营后台 |
| `display-app` | Web + Electron 桌面壳 | 班级大屏、白板展示、教师解锁操作 | 保持班级大屏定位，增加宇宙入口/主题展示 |

不建议在一期新增独立后端。新增前端建议命名为 `verse-web` 或作为 `display-app/web/public/verse` 的过渡型静态入口，最终应独立为学生/家长体验端。

### 2.2 已有数据能力

现有 Prisma 模型已经覆盖以下核心域：

- 学校、学期、年级、班级、教师、角色、权限、数据范围。
- 学生、学生成长档案、班级小组。
- 积分规则、学生积分记录、班级积分记录、小组积分记录、批量评价、撤销。
- 学业考试与成绩记录。
- 星宠、星宠阶段、学生星宠、星宠升级日志、星宠装扮。
- 荣誉、荣誉发放记录。
- 奖品、兑换订单。
- Display 终端、展示配置、教师解锁会话。
- AI 学情快照、教师观察。
- 教师课表、教师占用规则。
- 操作日志、叫号队列。

这些能力可直接支撑育英宇宙的一期核心闭环，不应重复建表或重写逻辑。

### 2.3 已有接口能力

现有接口前缀为 `/api/v1`，已具备：

| 能力 | 现有接口/模块 | 育英宇宙复用方式 |

| 星宠装扮 | `pet-decorations` | 数字生命卡、个性化装扮复用 |
| 积分记录 | `score-records`、`class-score-records`、`group-score-records` | 任务奖励、教师评价、成长档案复用 |
| 荣誉记录 | `honors`、`honor-records` | 荣誉广场、成长档案复用 |
| 奖励兑换 | `rewards`、`reward-orders` | 商城/背包/兑换中心复用 |
| AI 学情 | `ai/students`、Display academic AI | 家长成长报告、数字教师建议复用 |
| 课表 | `teacher-schedules` | 教学楼、教师办公楼、数字教师在线状态复用 |
| 操作日志 | `operation-log`、`audit` | 审核、溯源、校长信箱处理日志复用 |
| WebSocket | `realtime` | 大屏和宇宙端成长反馈实时刷新复用 |

## 3. 用户与场景

### 3.1 学生

核心诉求：

- 看到自己的星宠、等级、装扮、荣誉和最近成长。
- 在 2D 校园里探索建筑入口。
- 完成阅读、体育、劳动、文明、活动等任务。
- 看到自己和班级、小组的榜单位置。
- 与数字教师进行受控学习问答。

一期重点：学生体验闭环，不做复杂社交。

### 3.2 教师

核心诉求：

- 继续高效完成评价、加扣分、荣誉发放、任务布置。
- 在后台维护班级内容、任务、活动、书单。
- 通过 AI 学情和数字教师减轻重复答疑。
- 审核学生提交内容、校长信箱、数字教师问答风险。

一期重点：不改变教师高频工作流，宇宙任务优先映射到现有积分规则。

### 3.3 家长

核心诉求：

- 看懂孩子今天/本周在校成长。
- 查看星宠变化、荣誉、老师评价、阅读/体育/劳动等成长轨迹。
- 查看食堂菜单、校园活动和学校精选内容。

一期重点：提供只读成长窗口，不做家长即时聊天。

### 3.4 学校管理者

核心诉求：

- 展示 AI+教育、数字生命、校园元世界项目成果。
- 查看全校成长数据、荣誉数据和运营活跃度。
- 管理校长信箱、活动内容、数字教师安全边界。

一期重点：具备可演示、可汇报、可持续运营的产品框架。

### 5.4 一期不做

- 开放式多人实时移动、地图碰撞、即时聊天。
- 支付、电商、真实货币交易。
- 对外开放注册、多学校 SaaS 计费。
- 学生自由发布公开内容。
- 数字教师替代教师做评价或给出最终结论。
- 区块链、NFT、不可撤销存证。
- 复杂家校即时消息。

## 6. 2D 校园元世界需求

### 6.1 页面定位

2D 校园首页是育英宇宙的主入口。用户进入后应首先看到一个参考育英实验学校真实校园风貌的 2D 等距视角校园地图，并能快速识别核心建筑入口。

### 6.2 视觉风格

整体风格：

- 2D 等距视角或轻等距俯视。
- 明亮、清新、有校园真实感。
- 保留育英星宠红蓝校徽气质与荣誉感。
- 轻游戏化，但避免过度幼儿化、过度二次元化。
- 真实校园与“星球/宇宙”想象结合。

地图元素：

- 山地校园环境、白色现代建筑、大台阶、教学楼、运动场、校园广场。
- 星宠广场、校长办公室、海东大剧院、书吧、数字食堂、荣誉广场、教师办公楼、科技探索馆。
- 建筑热点有明确名称、状态和可点击反馈。

### 6.3 首页布局

| 区域 | 内容 |
|---|---|
| 顶部导航 | Logo、产品名、当前身份、星能/积分/徽章、消息、设置 |
| 左侧星宠面板 | 学生头像、姓名、班级、星宠、等级、经验条、荣誉、成长档案入口 |
| 中央地图 | 2D 校园地图、建筑热点、任务状态、活动标记 |
| 右侧信息栏 | 今日任务、校长信箱提示、校园活动、今日菜单、公告 |
| 底部导航 | 地图、任务、荣誉、背包、成长档案、家长空间（按角色显示） |

### 6.4 建筑需求

#### 星宠广场

定位：主城、出生点、星宠展示中心。

功能：

- 我的星宠高亮展示。
- 班级星宠墙。
- 星宠排行榜、成长榜、装扮主题入口。
- 主题活动入口。

数据来源：

- `student_pet`、`pet`、`pet_stage`、`student_profile`、`pet_decoration`。
- 排行榜复用 `GET /display/classes/:classId/leaderboard`。

#### 班级空间

定位：班级家园。

功能：

- 班级主页、班级成员、班级口号、班级目标。
- 小组积分、班级荣誉墙、班级任务。
- 班级成员星宠墙。

数据来源：

- `class`、`class_group`、`student`、`student_profile`、`score_record`、`class_group_score_record`。
- 复用 `GET /display/classes/:classId/home`、`GET /display/classes/:classId/roster`、`GET /classes/:id/groups`。

#### 荣誉广场

定位：荣誉展示与激励中心。

功能：

- 阅读之星、劳动之星、体育之星、文明之星、科技之星。
- 学生荣誉、集体荣誉、阶段荣誉。
- 星宠荣誉形象、班级荣誉榜。

数据来源：

- `honor`、`honor_record`。
- 复用 `GET /display/classes/:classId/honor-records`。

#### 书吧

定位：阅读成长中心，替代“数字图书馆”的冷入口。

功能：

- 推荐书单、阅读打卡、阅读挑战、阅读排行榜、阅读徽章、书香班级。
- 一期可由后台配置书单和挑战，完成确认后映射到现有积分规则。

新增需求：

- 内容模型：书单、推荐语、适用年级、封面、状态。
- 任务模型：阅读任务、周期、奖励规则、完成记录。

#### 校长办公室与校长信箱

定位：校园治理与学生心声中心。

硬性要求：

- 校长办公室必须是独立建筑，不只是右侧卡片或弹窗。
- 校长信箱设在校长办公室内。

功能：

- 校长寄语。
- 写信给校长。
- 信件分类：建议、表扬、问题、想对学校说、其他。
- 可选匿名提交。
- 处理状态：待处理、处理中、已回复、已关闭。
- 校长精选回复展示。

安全要求：

- 学生端仅查看自己的提交和精选公开回复。
- 后台可按权限查看、回复、隐藏、标记精选。
- 所有处理动作进入 `operation_log`。

#### 海东大剧院

定位：校园文化活动中心。

功能：

- 活动列表、活动详情、报名入口、作品展示、获奖展示。
- 一期以内容展示与轻报名为主，不做复杂票务或流程审批。

#### 教学楼

定位：课程学习与学业成长中心。

功能：

- 今日课程。
- 学习任务。
- 成绩成长入口。
- AI 学情摘要。

数据来源：

- `teacher_schedule_slot`、`academic_exam`、`academic_score_record`、`ai_student_snapshot`。

#### 教师办公楼

定位：数字教师入口。

功能：

- 教师列表、学科筛选、教师在线/授课状态。
- 基础学习问答、错题讲解、学习建议。
- 无法回答时提示咨询真人教师。

数据来源：

- `user`、`teacher_class_assignment`、`teacher_schedule_slot`。
- 数字教师配置和问答日志需新增。

#### 数字食堂（P1）

定位：校园饮食与健康空间。

功能：

- 今日菜单、本周菜单、营养结构、菜品评价、家长查看。
- 一期 P1 可先后台录入菜单，前端只读展示。

#### 科技探索馆（P1/P2）

定位：科技、AI、创客活动空间。

功能：

- AI 体验、科学实验、创客任务、科技徽章。
- 一期可作为内容入口，后续结合任务和作品提交。

## 7. 星宠数字生命体系

### 7.1 当前复用原则

星宠数字生命体系必须优先复用现有模型：

| 需求 | 现有模型/字段 |
|---|---|
| 星宠基础身份 | `pet.code`、`pet.name`、`student_pet.pet_id` |
| 学生绑定 | `student_pet.student_id` |
| 等级/阶段 | `student_pet.current_level`、`current_stage_no`、`pet_stage.level_no`、`stage_no` |
| 成长总分 | `student_pet.total_score`、`student_profile.total_score` |
| 当前积分 | `student_profile.current_score` |
| 星宠昵称 | `student_pet.nickname` |
| 升级历史 | `pet_level_log` |
| 装扮 | `pet_decoration`、`student_pet_decoration` |

原 PRD 中的 `exp` 不作为一期新增字段。星宠经验等同于现有成长积分/总分，避免和 `total_score` 产生双轨。

### 7.2 属性体系

一期属性值不新增独立实时账户表，先由积分规则中的 `dimension`、`tag`、`module_type`、`subject_code` 聚合计算。

| 属性 | 映射来源 |
|---|---|
| 知识值 | 学业、阅读、课堂、作业、答题相关规则 |
| 品格值 | 文明、德育、纪律、助人、责任相关规则 |
| 体能值 | 体育锻炼、运动、健康任务 |
| 创造力 | 艺术、科技、创作、展示活动 |
| 责任值 | 劳动实践、班级服务、小组贡献 |

后续如需高频查询，可新增 `verse_student_attribute_snapshot` 做周期快照。

### 7.3 成长反馈

每次成长反馈必须可溯源到真实记录：

```text
完成阅读挑战
↓
生成任务完成记录
↓
按配置调用现有积分规则生成 score_record
↓
更新 student_profile / student_pet
↓
如跨等级阈值，生成 pet_level_log
↓
WebSocket 推送前端成长反馈
```

### 7.4 数字生命卡

数字生命卡内容：

- 学生姓名、班级。
- 星宠名称、昵称、编号、当前等级、当前阶段形象。
- 当前积分、总成长值、属性雷达。
- 荣誉徽章、最近成长记录。
- 装扮组合。

数字生命卡不作为区块链证书。一期只做可分享/可截图/可打印展示。

## 8. 任务与运营体系

### 8.1 任务分类

| 类型 | 示例 | 一期处理方式 |
|---|---|---|
| 每日任务 | 阅读 30 分钟、文明行为、体育锻炼 | 后台配置，学生提交/教师确认 |
| 每周任务 | 阅读挑战、小组协作、班级目标 | 后台配置，按班级/年级发布 |
| 活动任务 | 艺术节、科技节、阅读节、体育节 | 与海东大剧院/科技探索馆/书吧联动 |
| 系统任务 | 首次进入、完善星宠昵称、查看成长档案 | 自动完成 |

### 8.2 奖励规则

任务奖励不另造积分账户，必须映射到：

- `score_rule`：定义奖励规则、分值、维度、标签。
- `score_record`：生成真实成长记录。
- `honor_record`：达到荣誉条件时发放荣誉。
- `reward_order` 或 `student_pet_decoration`：兑换奖品/装扮时复用现有链路。

### 8.3 任务完成状态

新增任务完成记录需至少包含：

- 任务 ID。
- 学生 ID、班级 ID、学校 ID。
- 状态：未开始、进行中、待确认、已完成、已驳回、已过期。
- 提交内容：文字/图片 URL 可选。
- 确认人、确认时间。
- 关联 `score_record_id`、`honor_record_id`。

## 9. 数字教师体系

### 9.1 一期定位

数字教师一期只做“受控学习助手”，不是开放聊天机器人，也不替代真人教师。

### 9.2 一期能力

- 教师/学科列表。
- 学科常见问答。
- 基础错题讲解。
- 学习建议。
- 基于现有 AI 学情快照给出阶段建议。
- 答不上或风险问题转真人教师。

### 9.3 管理后台需求

- 启用/停用数字教师。
- 配置教师头像、简介、学科、适用年级/班级。
- 配置知识库资料或 FAQ。
- 查看问答日志。
- 标记敏感/不当回答。
- 配置安全提示语。

### 9.4 安全边界

- 不回答与学习无关的敏感内容。
- 不生成不适合学生的内容。
- 不做医疗、法律、心理危机等高风险建议。
- 不直接给学生贴负面标签。
- 不给出最终成绩、处分、评价结论。
- 保留问答日志供教师和管理员审核。

## 10. 家长空间

### 10.1 一期定位

家长空间是“孩子今天过得怎么样”的只读窗口，不是家校即时聊天系统。

### 10.2 展示内容

- 孩子信息、班级、班主任。
- 星宠状态、等级、装扮、荣誉。
- 今日成长、本周成长、最近积分记录。
- 教师评价与教师观察摘要。
- AI 周/月成长摘要。
- 今日菜单、校园活动、阅读挑战。

### 10.3 数据安全

- 家长只能查看绑定孩子的数据。
- 家长绑定需后台审核或由学校批量导入。
- 默认隐藏其他学生姓名和敏感排行细节。
- 不向家长展示未审核的学生来信、内部操作日志、教师内部备注。

## 13. 权限与安全

### 13.1 角色复用与新增

复用现有角色：

- `super_admin`
- `school_admin`
- `moral_admin`
- `grade_admin`
- `homeroom_teacher`
- `subject_teacher`
- `display_account`

建议新增：

- `student_account`
- `parent_account`
- `principal_account` 或通过 `dutyTags` 标记校长/管理职务。

### 13.2 权限原则

- 学生只能访问本人数据、所在班级公开内容和公开榜单。
- 家长只能访问绑定学生的数据。
- 班主任可管理本班任务、查看本班提交。
- 任课教师按 `teacher_class_assignment` 和 `user_scope` 控制可见班级。
- 校长信箱匿名只对前台匿名，后台高权限可按审计策略保留必要溯源。
- 数字教师问答日志不得对普通学生公开。

### 13.3 隐私与未成年人保护

- 不公开学生敏感负面记录。
- 榜单展示需支持弱化姓名或仅班内可见。
- 家长端默认只展示正向成长和教师确认过的信息。
- 图片/附件上传需限制类型、大小并做审核。
- AI 输出必须经过安全过滤和日志留存。

## 15. 关键验收标准

### 15.1 产品闭环验收

- 学生能进入 2D 校园首页。
- 能看到真实学生、班级、星宠、等级、积分和荣誉数据。
- 至少 6 个 P0 建筑可点击并进入详情。
- 至少 1 个任务可提交、教师可确认、确认后生成真实积分记录。
- 星宠成长、成长档案、榜单能看到对应变化。
- 校长信箱可提交、后台可回复、学生可查看回复。
- 数字教师可选择教师/学科并完成一次安全问答。

### 15.2 数据一致性验收

- 任务奖励生成的成长记录必须写入 `score_record`，不得只存在前端 mock。
- 星宠等级展示必须来自 `student_pet` / `pet_stage`，不得单独硬编码。
- 荣誉广场必须读取 `honor_record`。
- 家长空间必须受绑定关系限制。
- 后台处理动作必须可在操作日志或对应业务日志中追溯。

### 15.3 UI 验收

- 1920x1080 大屏下地图、建筑名称、星宠侧栏、右侧任务区不重叠。
- 平板/手机下核心入口可访问，文字不溢出按钮或卡片。
- 地图资源加载失败时有降级占位。
- 大屏低配设备可关闭或降低动画。

### 15.4 安全验收

- 未登录用户不能访问个人成长数据。
- 学生不能访问其他学生私密成长档案。
- 家长不能访问未绑定学生。
- 任课教师不能越权确认非授权班级任务。
- 数字教师敏感输入有拒答或转人工提示。

## 18. 最终产品边界

育英宇宙一期的成功标准不是“像游戏一样复杂”，而是：

- 真实接入育英星宠现有数据。
- 让星宠和成长数据变得更可感知。
- 让校园建筑成为服务和内容入口。
- 让任务、荣誉、信箱、数字教师能够被后台运营。
- 让学校具备可展示、可持续迭代的 AI+教育元世界项目。

因此，一期必须坚守：

```text
复用现有业务底座
新增体验入口和少量运营域
所有成长反馈回写真实数据
所有学生/家长可见数据受权限保护
所有 AI 与信箱内容可审核可追溯
```
