# 育英星宠 PRD

版本：V1.0  
日期：2026-03-30  
产品名称：`育英星宠`

## 一、项目背景

`育英星宠` 是一个专为育英学校开发的高度定制化班级积分管理系统，目标不是做通用 SaaS，而是做一套能够同时服务于：

- 班级日常激励
- 学校德育管理
- 领导视察展示
- 校园品牌呈现

的双端系统。

结合前期研究，参考系统 `bancxq.com` 在“班级萌宠成长 + 学生卡片 + 榜单激励”方面有较成熟的产品骨架，但其整体仍偏“班级运营工具”，不足以满足育英学校对“学校专属感、仪式感、汇报展示感”的要求。

因此，`育英星宠` 的定位明确为：

`校园荣誉体系下的萌宠成长激励平台`

## 二、产品目标

## 1. 业务目标

- 建立统一的班级积分激励机制
- 通过萌宠成长与荣誉展示提高学生参与度
- 帮助班主任更高效地开展班级评价
- 为学校提供可视化、可汇报的德育数据看板
- 在白板/大屏场景下形成有仪式感的校园展示体验

## 2. 产品目标

- 打造学校专属视觉风格，体现育英学校品牌
- 搭建 `admin` 管理后台与 `display` 班级展示端双端体系
- 在产品结构上形成“评价 -> 成长 -> 榜单 -> 荣誉 -> 数据”的完整闭环
- 优先满足学校定制化需求，而非追求通用能力

## 三、产品定位

`育英星宠` 不是普通积分工具，也不是幼儿化萌宠应用，而是：

`带有学校荣誉气质的校园成长系统`

关键词：
- 学校定制
- 班级激励
- 萌宠成长
- 荣誉体系
- 校级驾驶舱
- 白板展示

## 四、产品范围

本期产品范围包含两个端：

## 1. Admin 管理后台

使用方式：Web  
主要使用者：学校管理员、德育负责人、班主任、相关教师

主要职责：
- 学校基础信息管理
- 班级与学生信息管理
- 积分规则管理
- 荣誉/勋章管理
- 奖励中心管理
- 萌宠图鉴配置
- 数据统计与校级驾驶舱展示

## 2. Display 班级展示端

使用方式：SPA Web，全屏运行于 Windows 白板、触控一体机或浏览器大屏  
主要使用者：班主任、学生

主要职责：
- 学校品牌进入页展示
- 班主任登录
- 进入教学楼/班级过场动画
- 班级宠物主页展示
- 学生萌宠卡片展示
- 榜单轮播、荣誉展示、班级目标展示

说明：
- 原“Windows 应用程序”需求已统一收敛为 `运行在 Windows 设备上的展示型 SPA`
- 不单独开发原生桌面端，后续如有需要可再封装为桌面壳

## 五、目标用户

## 1. 学校管理员

关注点：
- 学校整体使用情况
- 年级/班级活跃度
- 数据汇总与成果展示
- 学校品牌形象

核心诉求：
- 能看全校数据
- 能给领导看
- 能支撑校内管理

## 2. 德育负责人 / 年级负责人

关注点：
- 正向行为分布
- 班级差异
- 荣誉与激励机制执行情况

核心诉求：
- 规则清晰
- 数据可追踪
- 有阶段成果展示能力

## 3. 班主任

关注点：
- 课堂即时评价是否方便
- 班级激励是否有趣
- 学生是否愿意参与

核心诉求：
- 操作简单
- 展示效果强
- 奖惩反馈直观

## 4. 学生

关注点：
- 自己的萌宠是否成长
- 是否能上榜
- 是否能获得荣誉和奖励

核心诉求：
- 看得见成长
- 有荣誉感
- 有参与动力

## 六、核心设计原则

- 学校感优先：整体视觉围绕育英学校红蓝校徽体系构建
- 展示感优先：支持领导视察、家长参观、校园展示场景
- 班级高频优先：班主任高频操作必须足够轻
- 成长感优先：积分变化必须能映射到萌宠和荣誉变化
- 分层清晰：管理后台与展示端职责严格区分

## 七、产品视觉方向

基于 `logo.png` 与 `display-bg.png` 提炼：

- 品牌主蓝：学校权威感、秩序感、基底色
- 品牌强调红：识别感、仪式感、重点强调
- 白色线稿：校园建筑、校训、空间感
- 金色：荣誉、勋章、榜单高光

视觉关键词：
- 校徽感
- 徽章感
- 展板感
- 校园数字化
- 宠物成长感

避免：
- 过度幼儿化
- 通用后台模板感
- 花哨炫技式大屏

## 八、核心业务闭环

系统核心闭环如下：

`教师评价 -> 学生得分 -> 萌宠成长 -> 榜单排名 -> 荣誉/奖励 -> 数据沉淀`

拆解说明：
- 教师在班级场景中给学生进行即时评价
- 学生获得积分、经验、勋章或标签
- 萌宠形象、等级、成长状态发生变化
- 榜单与班级高光内容被展示
- 达成条件后可获得荣誉或奖励
- 所有行为沉淀为班级、年级、学校数据

## 十、核心功能需求

## A. Admin 管理后台

## A1. 校级驾驶舱

### 功能目标

用于学校管理层、德育负责人、领导视察等场景，集中展示学校关注的核心数据。

### 核心需求

- 展示全校总积分、活跃班级数、学生总数、勋章发放量等核心指标
- 支持按学段、年级、班级查看数据
- 展示近期活跃趋势、行为趋势、成长趋势
- 展示明星班级、优秀学生、班级排行
- 展示重点预警信息，如低活跃班级、异常数据
- 支持汇报模式展示

### 产品要求

- 首页视觉要有“学校驾驶舱”质感
- 不可做成普通后台卡片拼接
- 图表需要整洁、正式、可汇报

## A2. 班级管理

### 核心需求

- 创建/编辑班级
- 配置年级、班主任、班级口号、班级目标
- 配置 display 端对应班级展示页
- 设置班级分组或展示顺序

## A3. 学生管理

### 核心需求

- 学生信息录入、导入、编辑
- 绑定学生所属班级
- 绑定学生萌宠形象
- 查看学生积分、等级、勋章、成长记录

## A4. 积分规则管理

### 核心需求

- 配置加分项、减分项、分值
- 支持按行为分类管理
- 支持启用/停用规则
- 支持班级/年级差异化规则扩展

## A5. 荣誉勋章管理

### 核心需求

- 配置勋章名称、图标、获取条件、展示文案
- 区分个人荣誉与集体荣誉
- 支持阶段荣誉、长期荣誉

## A6. 奖励中心

### 核心需求

- 配置奖励项
- 配置兑换门槛或达成条件
- 记录兑换/获得情况

说明：
- 学校定制版不应只强调物质奖励
- 需支持精神奖励、荣誉称号、班级特权等形式

## A7. 萌宠图鉴

### 核心需求

- 维护萌宠基础资料
- 配置萌宠成长阶段
- 管理萌宠展示图与素材
- 支持按系列/类型筛选

## A8. 数据分析

### 核心需求

- 班级维度趋势分析
- 年级维度横向对比
- 学校维度汇总分析
- 榜单导出、图表导出

## A9. 系统设置

### 核心需求

- 学校品牌配置
- 校训、校徽、学校图片管理
- 学期配置
- 角色权限管理
- Display 进入页配置

## B. Display 班级展示端

## B1. 进入页

### 功能目标

进入页承担学校品牌展示与班级进入前的过渡作用。

### 设计要求

- 使用 `display-bg.png` 作为进入页主背景
- 展示校训、校徽、学校名、产品名 `育英星宠`
- 整体画面体现学校专属感与仪式感

### 页面元素

- 顶部：校训文字
- 中部：校徽 + 产品名 + 副标题
- 下方：班主任登录入口
- 底部：教学楼线稿

## B2. 班主任登录

### 核心需求

- 支持班主任账号登录
- 登录成功后自动进入对应班级
- 支持记住登录态

### 安全要求

- 展示端不暴露复杂管理能力
- 展示端仅提供班级展示相关权限

## B3. 进入校园/教学楼/班级动画

### 功能目标

形成区别于普通系统的开场仪式感。

### 动画脚本要求

- 登录后先从学校主视觉进入
- 镜头沿教学楼或台阶方向推进
- 最终进入班级主界面

### 时长要求

- 完整版：6-8 秒
- 快速版：2-3 秒
- 支持跳过

## B4. 班级主页

### 功能目标

班级主页是 Display 端核心页面，承担日常白板展示与课堂激励展示。

### 页面结构

- 顶部：班级名、日期、班级口号、荣誉状态
- 中区：学生萌宠卡片矩阵
- 侧边：排行榜 / 今日之星 / 班级目标
- 底部：班级总积分 / 最近表扬 / 滚动通知

### 核心需求

- 展示学生姓名、宠物、等级、积分、勋章等信息
- 支持大屏分辨率适配
- 支持自动刷新
- 支持课堂中实时反馈变化

## B5. 榜单页

### 核心需求

- 展示经验榜、勋章榜、小组榜等
- 支持前 3 名高光展示
- 支持日榜、周榜、月榜切换扩展

## B6. 荣誉展示页

### 核心需求

- 展示近期获得的荣誉
- 展示班级荣誉墙
- 支持活动期专题展示

## B7. 班级目标页

### 核心需求

- 展示本周/本月班级目标
- 展示完成进度
- 展示集体成就

## B8. 教师控制页（可选）

### 目标

将展示页与高频控制行为分离，避免在大屏正面暴露过多操作按钮。

### 核心需求

- 快速加减分
- 切换展示模式
- 控制轮播
- 触发荣誉动画或奖励反馈

## 十二、关键流程

## 1. 班主任进入班级展示流程

1. 打开 Display 端进入页
2. 展示学校主视觉与校训
3. 班主任登录
4. 播放“进入校园 -> 进入教学楼 -> 进入班级”动画
5. 进入对应班级主页

## 2. 学生日常激励流程

1. 教师发起评价
2. 学生获得积分/经验/勋章
3. 班级主页实时更新
4. 榜单与荣誉内容发生变化
5. 学生获得成长反馈

## 3. 管理层查看数据流程

1. 登录 Admin
2. 进入校级驾驶舱
3. 查看学校整体与分层数据
4. 进入详细分析页或榜单页
5. 在汇报场景下进行展示

## 十三、数据指标建议

## 1. 校级驾驶舱重点指标

- 全校总积分
- 活跃班级数
- 活跃学生数
- 本周正向行为次数
- 年级参与率
- 勋章发放数
- 班级平均成长等级
- 明星班级
- 明星学生

## 2. 班级展示端重点指标

- 班级总积分
- 今日加分次数
- 今日之星
- 班级目标完成度
- 当前榜单

## 十四、角色与权限

建议角色：
- 超级管理员
- 学校管理员
- 德育负责人
- 年级负责人
- 班主任
- 任课教师
- 展示端账号

权限原则：
- Admin 拥有配置与管理权限
- Display 仅拥有展示与轻量控制权限
- 高风险配置必须进入后台完成

## 十五、非功能需求

## 1. 视觉要求

- 必须体现育英学校专属视觉风格
- 使用红蓝主色体系
- Display 端需有明显的品牌仪式感

## 2. 性能要求

- Display 首屏加载快速
- 班级主页刷新流畅
- 大屏场景下动画不卡顿

## 3. 兼容性要求

- Admin 兼容主流现代浏览器
- Display 兼容 Windows 浏览器、触控一体机全屏场景
- 支持常见 1080P 横屏显示

## 4. 安全要求

- 展示端默认弱化管理权限
- 关键管理行为需鉴权
- 生产环境需记录核心操作日志

## 十六、与参考系统对比后的策略结论

参考系统值得借鉴的部分：
- 学生卡片式班级主页
- 萌宠成长驱动
- 榜单与奖励闭环

参考系统不宜直接照搬的部分：
- 数据仪表盘更偏班级运营，不够校级汇报
- 学校品牌感不足
- 展示端缺少学校主视觉入口和教学楼过场逻辑

因此，`育英星宠` 的产品策略是：

- 借鉴其“班级激励骨架”
- 强化“学校品牌与仪式感”
- 增加“校级驾驶舱”
- 重做“展示端进入体验”

优先做出的最小闭环：

1. Admin 校级驾驶舱
2. 班级与学生管理

## 十八、成功标准

产品上线后，可从以下角度评估是否成功：

- 班主任是否愿意高频使用
- 学生是否对萌宠成长与榜单有明显关注
- 学校是否能在视察、汇报时使用首页展示
- 学校是否认可该系统具有“育英学校专属感”

## 十九、结论

`育英星宠` 的核心不是“做一个积分系统”，而是：

`把学校德育激励、班级成长反馈、校园品牌展示整合为一套可感知、可展示、可管理的数字化系统。`

它的关键差异化在于：
- 学校专属视觉
- 班级宠物成长体验
- 校级驾驶舱
- 进入校园的展示仪式感

这四点同时成立，产品才会真正形成辨识度。
