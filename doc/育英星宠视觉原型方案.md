# 育英星宠视觉原型方案

适用范围：
- 产品名：`育英星宠`
- `admin` 管理后台
- `display` 班级展示端
- 视觉基准来源：`logo.png` 与 `display-bg.png`

## 一、视觉基因分析

## 1. 从 `logo.png` 提炼出的品牌特征

品牌气质不是卡通软萌路线，而是：
- 学校荣誉感
- 徽章感
- 仪式感
- 秩序感

核心视觉元素：
- `深蓝色盾牌`
- `高饱和红色飘带`
- `皇冠`
- `星形`
- `校徽式中轴构图`

这意味着 `育英星宠` 虽然是萌宠积分系统，但不能做成普通儿童产品风格。更合适的方向是：

`校园荣誉体系 + 成长宠物系统`

也就是：
- 宠物负责亲和力与成长感
- 校徽红蓝负责学校气质与品牌感

## 2. 从 `display-bg.png` 提炼出的展示端气质

这张图的价值非常高，适合直接作为 `display` 端进入页底图。

它传达出的感觉是：
- 庄重
- 校园专属
- 有空间纵深
- 适合做“进入校园”的过场

关键元素：
- 大面积深蓝背景
- 白色校训文字
- 中部校徽
- 底部教学楼线稿
- 楼前阶梯天然适合作为“镜头推进路径”

因此，`display-bg.png` 不应该只被当作静态背景，而应该被当作：

`育英星宠 display 端的品牌入口场景`

## 二、视觉总方向

整体建议采用：

`校徽荣誉风 + 数字校园风 + 宠物成长感`

避免：
- 过于幼儿化的糖果色
- 泛互联网后台的蓝白模板
- 紫色系、渐变泡泡系、二次元系

建议色板如下：

```text
品牌主蓝：#0F3D63
品牌深蓝：#0A2F4B
品牌强调红：#E64626
品牌深红：#C92F18
辅助白：#F7FAFC
线稿灰蓝：#88A3B7
荣誉金：#D7A441
成功绿：#20A36A
警示橙：#F38B2A
```

使用原则：
- 蓝色负责大面积基底、结构层次、学校权威感
- 红色负责按钮、徽章、关键数字、当前态
- 金色只用于荣誉、排行、校级成果高光

## 三、Logo 延展建议

产品名已经明确为：`育英星宠`

建议品牌字标结构：
- 主标题：`育英星宠`
- 副标题：`YUYING STARPET`

建议图形组合：
- 保留盾牌/星形/冠冕语义
- 在盾牌内部加入“星球轨道”或“宠物爪印”作为次级识别
- 不要完全脱离校徽体系，否则会失去学校定制感

建议形成两套标识：
- 完整版：校徽语义 + 育英星宠字标
- 应用版：简化盾牌星宠图标，用于导航栏、启动页、favicon

## 四、Admin 端原型方向

## 1. 首页定位

`admin` 首页不要做普通后台首页，而要做：

`校级驾驶舱首页`

布局建议：

```text
顶部：学校品牌栏 + 日期天气 + 当前学期 + 快捷入口
第一屏：校级关键指标
第二屏：年级/班级分布与趋势
第三屏：高光榜单与荣誉展示
第四屏：重点预警与待处理事项
```

## 2. 首页视觉结构

建议采用“深蓝主背景 + 白色卡片 + 红色高亮”的结构。

页面视觉重点：
- 顶部横幅使用红蓝叠层与细线星轨
- 卡片不要纯圆角 SaaS 风，要带一点“徽章/展板”感
- 重要数字区域用大号数字 + 小标签 + 环形/柱状辅助图

建议首屏卡片：
- 全校总积分
- 活跃班级数
- 本周正向行为次数
- 年级参与率
- 勋章发放数
- 本月明星班级

## 3. Admin 页面的设计语言

导航建议：
- 左侧深蓝纵向导航
- 当前页使用红色短条 + 浅蓝底高亮
- 图标线条风格统一，避免默认后台图标库味道过重

图表建议：
- 主图表尽量用深蓝、浅蓝、红、金四色体系
- 不做过花的霓虹大屏，而是做“学校汇报级整洁图表”
- 榜单与荣誉区允许更强视觉表现

## 4. Admin 主要页面原型清单

建议页面：
- `首页 / 校级驾驶舱`
- `班级管理`
- `学生管理`
- `积分规则`
- `勋章与荣誉`
- `奖励中心`
- `萌宠图鉴`
- `数据分析`
- `系统设置`

其中最需要重点设计的是：
- `首页 / 校级驾驶舱`
- `数据分析`
- `勋章与荣誉`

## 五、Display 端原型方向

## 1. 进入页定义

`display-bg.png` 直接作为 `display` 端进入页背景。

页面结构建议：

```text
顶部：校训文案
中部：校徽 / 育英星宠品牌标题 / 班主任登录入口
底部：教学楼线稿 + 阶梯推进动效
```

页面文案建议：
- 主标题：`育英星宠`
- 副标题：`育英学校班级成长激励系统`
- 操作提示：`请班主任登录，开启班级星宠成长之旅`

## 2. 进入页动效脚本

建议做成三段式：

### 第一段：校园待机
- 展示 `display-bg.png`
- 校训与校徽静态呈现
- 背景做极轻微呼吸动效
- 教学楼线稿可以有细微流光

### 第二段：登录后进入校园
- 点击登录后，中央品牌区渐隐
- 镜头沿楼前阶梯向前推进
- 建筑线稿逐渐放大，模拟“走进校园”

### 第三段：进入班级
- 画面由校园蓝色场景切入班级主界面
- 班级名、老师名、班级口号先出现
- 随后学生宠物卡片分批登场

动画总时长建议：
- 完整版：`6-8 秒`
- 日常快速版：`2-3 秒`

## 3. Display 首页原型

进入动画结束后，主界面建议是：

```text
顶部：班级名称 / 今日口号 / 日期 / 荣誉状态
中区：学生星宠卡片矩阵
右侧：今日排行榜 / 班级目标 / 今日之星
底部：班级积分总览 / 最近表扬 / 滚动通知
```

视觉原则：
- 保留学校红蓝主色，不要变成普通卡通乐园
- 宠物卡片可以更鲜活，但背景要稳重
- 可加入星轨、徽章、校园轮廓线作为装饰母题

## 4. Display 卡片样式建议

每张学生卡包含：
- 学生姓名
- 宠物形象
- 当前等级
- 本周积分
- 勋章数
- 正向标签

卡片风格建议：
- 上半部展示宠物大图
- 下半部是信息铭牌
- 卡片边框可借鉴校徽盾牌轮廓
- 排名前列学生卡可加入金色荣誉边框

## 六、两端统一的组件风格

建议统一这几类组件：
- `荣誉卡`
- `排行榜卡`
- `萌宠卡`
- `数据指标卡`
- `勋章组件`
- `校徽风弹窗`

统一细节：
- 圆角不要太大，建议偏稳重
- 边框与阴影要克制
- 标题字尽量有“校务展板”的秩序感
- 红色只用于关键强调，不要满屏铺红

## 七、原型落地建议

如果直接进入设计或前端阶段，建议优先做 4 个页面原型：

1. `Admin 校级驾驶舱首页`
2. `Display 进入页`
3. `Display 班级主页`
4. `Admin 萌宠图鉴页`

这 4 页一旦成型，`育英星宠` 的品牌调性、学校感和产品骨架就基本确定了。

## 八、结论

`育英星宠` 不应该设计成普通萌宠积分应用，而应该设计成：

`带有学校荣誉气质的校园成长系统`

视觉上以：
- `深蓝` 建立学校权威感
- `红色` 建立品牌识别与仪式感
- `宠物成长` 承担亲和力与参与感

其中：
- `logo.png` 决定品牌语义
- `display-bg.png` 决定展示端入口场景

这两张图已经足够成为整个产品原型的视觉母版。
