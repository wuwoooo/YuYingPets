# 育英星宠数据库表结构设计

版本：V1.0  
日期：2026-04-14  
适用范围：`YuYingPets` 项目第一阶段开发

## 1. 设计目标

本设计用于支撑以下核心能力：

- 学校、学期、班级、学生基础管理
- Admin / Display 双端统一加减分
- 萌宠成长与等级展示
- 奖励兑换
- 荣誉勋章
- 实时榜单与班级展示
- AI 学情评估的数据沉淀

数据库选型已确定为：`MySQL 8+`

## 2. 设计原则

- 所有核心业务实体使用自增主键 `BIGINT`
- 所有时间字段统一保留：
  - `created_at`
  - `updated_at`
- 重要业务表保留软删除字段：
  - `deleted_at`
- 状态值使用 `VARCHAR` 或 `TINYINT`
- 尽量避免把业务语义塞进 JSON
- 为 AI 学情分析预留结构化字段，不依赖后期猜测

## 3. 命名约定

- 表名使用小写下划线
- 主键统一为 `id`
- 外键字段使用 `<entity>_id`
- 布尔值尽量使用 `is_xxx`
- 编码字段尽量使用 `code`

## 4. 核心 ER 关系概览

```text
school
  ├─ semester
  ├─ user
  ├─ role
  ├─ class
  │    ├─ class_group
  │    ├─ student
  │    │    ├─ student_profile
  │    │    ├─ student_pet
  │    │    ├─ student_group_rel

  │    └─ display_config
  ├─ score_rule

  │    └─ pet_stage
  ├─ reward
  └─ honor
```

- `score_rule` 仅支持固定分值
- 原 `.xls` 中的区间分值规则，在导入时先拆分成多条固定规则或选定固定档位
